# 更新履歴

VaultOne の主な変更点をすべて記録します。

形式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に基づき、本プロジェクトは[セマンティックバージョニング](https://semver.org/lang/ja/)に従います。

## [1.1.0] - 2026-07-23

### 追加

- **自動更新** —— 起動時に新バージョンを確認（24 時間に 1 回までスロットル）、または設定から手動確認。署名付きインストーラーを GitHub Releases から直接ダウンロード・インストールし、Ed25519 署名検証とワンクリック再起動に対応。GitHub 経由で完全に配信——セルフホストサーバー不要。更新失敗時は Releases ページを開く手動フォールバック。
- **表示言語** —— UI を English / 简体中文 / 日本語 で切り替え。

### 修正

- **ライトウェイトモード** —— 格納された覗き見アイコンのエッジ吸着と、斜め方向の展開アニメーションを滑らかに。

## [1.0.0] - 2026-07-23

初の公開リリース（オープンソース）。

### 追加

- **ダッシュボード** —— 4 バケットのトークン消費（input / output / cache creation / cache read）、キャッシュヒット率（`cache_read / (input + cache_creation + cache_read)`）、総リクエスト数と総コスト（USD、収集時に固定）、トークン対コストの双軸使用量トレンド、コールごとのリクエストログ（モデル・トークン内訳・コスト・ターン所要時間・`stop_reason` / `service_tier` チップ）、ターンごとのコストと実所要時間ビュー。
- **収集** —— Claude Code セッションログの読み取り専用解析（ソースログは変更しない）、カーソルベースの増分スキャン、トレイ常駐のバックグラウンドスケジューラ。プラグイン可能なプロバイダアーキテクチャ（現在は Claude Code、追加予定）。
- **同期（任意）** —— スタンドアローンモード（完全なダッシュボード、ネットワークゼロ）と同期モード（自分が管理する GitHub リポジトリ経由で端末間の使用量を整合）。プレーンテキストのアーティファクトを端末と日付で分割（`data/<device>/usage-YYYY-MM-DD.jsonl`）。
- **コストと価格** —— 編集可能なモデルごとの価格上書き。収集時に価格が無かったレコードの再請求（既存履歴を再計算しない）。
- **体験** —— ライトウェイトモード（エッジ格納 + ホバーで今日の使用量をちらっと確認）、カスタムタイトルバー、ライト / ダークテーマ、ローカルファーストでデフォルトプライベート。
- **パッケージング** —— Windows、macOS（Apple Silicon）、Linux 向けクロスプラットフォームインストーラ。タグ push で GitHub Actions が自動ビルド。

### 既知の制限

- **macOS**: Apple Silicon（arm64）のみ。ビルドは未署名——初回起動時に右クリック → **開く**（または `xattr -dr com.apple.quarantine /Applications/VaultOne.app`）。Intel Mac ユーザーはソースからビルド可能。
- **プロバイダ**: 現在は Claude Code のみ。追加プロバイダ（Codex、Cursor など）を計画中。
